Information Infrastructures

Heterogenous Information Infrastructure

A collection of diverse IT systems can be heterogenous in different ways:
  • - Data format: syntactic heterogenity
  • - Meaning of data: semantic heterogenity (some concepts have different eanings (content) in different systems)
  • - Data structure: structural heterogenity
  • - Programming languages
  • - System heterogenity: e.g. hardware, operating system

Reason: The collection of systems has 'grown' over time
E.g. organization has changed

Legacy Systems

Old systems/technology/applications that are still in use because:

  • It contains important and valuable data
  • It works fine
  • It is too costly to change, it must work 24/7
What is the problem with old systems?
  • Missing documentation/knowledge of system
  • Not flexible: one cannot add new data fields, new user area, adapt to an organization's evolving work processes etc.
  • Costly to mantain
  • Usually designed as "silos" of information
How to live with legacy systems?

Different strategies can be drastic (ripe and replace) or evolutionary (enhance and evolve)


Evolutionary - enhance and evolve
  • Code refactoring - restructuring existing code by changing the 'inside' structure but not its external behavior. This requires software archeology and reverse engineering
  • 'Wrapping'
  • Build a datawarehouse on top
  • 'Migrating'
Standard systems

Standard software packages

  • Enterprise resource planning (ERP) systems
  • E.g. SAP suite
