Types of Machine learning
Machine Learning

The ability of a program to learn from experience - that is, to modify its execution on the basis of newly acquired information.

Machine learning is about automatically extracting relevant information from data and aqpplying it to analyze new data.

Generalization: Provide sensible outputs for inputs not encountered during training.

The learning process is iterative.

Supervised Learning

Supervised learning is the machine learning task of determining a function from labeled data. For example, in a machine learning algorithm that detects if a post is spam or not, the training set would include posts labeled as "spam" and posts labeled as "not spam" to help teach the algorithm how to recognize the difference. Supervised learning algorithms infer a function from labeled data and uses this function on new examples. Since the algorithm receives a data set as input that already has a correct answer associated with it (target), the algorithm will learn by comparing its output with the correct answer, and if it finds errors, it adjusts the model accordingly. The training continues untiles the algorithm outputs information within the desired accuracy range.

We need a well defined set of observations, O (possible inputs) and a set of target values, T (possible output values).
The goal is to determine a mapping, Y, from O to T. We will acquire a training set of examples from OxT. The training phase will consist of Y learning from the training set. Ideally we want to have Y(o_j) = t_j on the training data.
When Y is learned, it can be used to predict values for new items, y(o).

Risks

In determining how good a specific supervised learning algorithm is, two types of risks can be minimized:

- Empirical risk: Empirical is the expected loss of the function y that the supervised learning algorithm infers from the traiing dataset. For example, if y correctly maps all training data points o_i, to their respective labels t_i, the empirical risk is 0.

In minimizing empirical risk, the supervised learning algorithm is taught to match the training data as well as possible. However, as shown in the image below, a solution can minimize empirical risk without being a good candidate function for unknown data points. This is called overfitting, which occurs when the proposed function focuses more on noise rather than the actual data, as seen below with the blue line.

For the given set of red input points, both the green and blue lines minimize error to 0. However the green line may be more succesful at predicting the coordinates of unknown data points, since it seems to generalize the data better.

- Structural risk: Structural risk is used to prevent the supervised learning algorithm from overfitting the training data. Structural risk minimiaztion introduces a regularization penalty that can prefer certain solutions over others.

A good way of determining an appropriate value for (lambda) is by using cross validation, a method that trains the supervised algorithm on training data and tests its performance on a validation dataset (data points where the correct label is known). The algorithm is then updated to mnimize its error on the validation set, while still being trained on the training dataset. An example of this in action is described in ridge regression, and is widely used to determine functions that perform well on known data.

Challenges
  1. Bias-variance tradeoff - Suppose a supervised learning algorithm is trained on multiple datasets. If the algorithm is unable to correctly label a specific data point, it is said to be biased for that input. Additionally, if the algorithm produces different output values when trained on different datasets, it is said to have high variance. Empirical risk focues on bias, whereas structural risk focuses on variance. There is usually a tradeoff between bias and variance, where a low bias implies high variance and vice-versa. A problem for supervised algorithms is finding the balance between these two concepts that works best with unknown data points.
    The blue curve minimizes the error of data points (low bias) but has high variance. Conversely, the black line does not minimize the errors (has a higher bias), but fits the data well (low variance).
  2. Complexity - The function that the supervised learning algorithm is trying to mimic could be either simple or complex. If the expected function is simple, the algorithm should have low variance to fit the data well. However, if the expected function is complex, the algorithm should have high variance to adapt to unknown data points. Supervised learning algorithms should be able to determine variance appropriately based on the amount of data and the type of function to be expected.
  3. Many dimensions - When a supervised learning algorithm is given a dataset consisting of many dimensions, it may attempt to identify trends between irrelevant factors. This increases the variance of the inferred function and can decrease the accuracy of the algorithm. Two ways of combating this issue includes running a different algorithm to discard irrelevant variables and reducing the input data to a lower number of dimensions.
  4. If the training data has errors in its labeling or in its data value, the supervised learning algorithm should not attempt to exactly match the training examples. This can lead to overfitting and will not perform well for unknown values. Additionally, if the training data contains redundant information, supervised learning algorithms may perform poorly due to overrelying on specific examples. Filtering the data or regularizing the algorithm properly can mitigate these two issues.

kNN - k Nearest Neighbors algorithm
  1. Calculate the distance to all the training instances
  2. Pick the k nearest ones
  3. Choose the majority class for this set
Distance

The assumption is based on points that are close together in feature space are similar. There are some notion of distance in feature space, normally Euclidean distance (L2).

More classes

A binary classifier with odd k always reaches a decision. With more than 2 classes, there might be a draw.
Possible ways out:

  • Alt. 1: Back off to k-1, etc. until there is a majority class
  • Alt. 2: Weight classes by distance from target, take the weighted max.

kNN with varying k

Don't train on test data. It can be legitimate to test performance on train data and compare to test data for overfitting.
The best k differ from task to task.

Determine k

To determine the best k for a specific task, split your development set in:

  • One set for training
  • One set for tuning k

Test for various ks on the tuning set. Choose the best performing k for your model. When your have determined k, you may test with this k on the final test set.
Dont use the same set for tuning as for final evaluation.
